## 已解决的认证问题

原问题：API 返回 `{"code":1,"msg":"无效的票据"}`

根本原因有三：

### 1. Authorization 头格式错误

浏览器实际发送的是 `Authorization: BasicAuth <ticket>`（一个词 `BasicAuth`），而非 `Authorization: Basic <ticket>`。

### 2. 缺少 token 签名

慢慢买 `customRequest.js` 中有反爬签名机制。API 请求需要额外的 `t`（时间戳）和 `token`（MD5 签名）参数。

签名算法：
```
secret = 'c5c3f201a8e8fc634d37a766a0299218'  // 从 customRequest.js RC4 解密得到
params.t = Date.now()
str = secret + encodeURIComponent(sortedKey1) + encodeURIComponent(sortedVal1) + ... + secret
str = str.toUpperCase()
token = MD5(str).toUpperCase()
```

### 3. 缺少 ASP.NET_SessionId 等会话 Cookie

脚本已实现：GET 请求响应中自动捕获 `Set-Cookie` 头合并到 Cookie 字符串中。

## 密钥更新机制

如果慢慢买更新了 `customRequest.js` 导致密钥失效，可通过设置弹窗底部的「更新密钥」按钮一键重新提取：

```javascript
GM_xmlhttpRequest GET https://mmbres.manmanbuy.com/pc_tool/customRequest.js
  → 正则提取 __0xa3e63 混淆数组
  → 执行 476 次 shift 洗牌
  → RC4 解密 elements[0]（密钥 'teCD'）
  → 得到 secret 并存入 GM_setValue
```

## 数据获取流程

```
用户提供慢慢买 Cookie → 存 GM_setValue
  → GM_xmlhttpRequest GET 慢慢买页面（tool.manmanbuy.com/HistoryLowest.aspx?url=京东商品链接）
    → 自动捕获 Set-Cookie 合并到 Cookie
    → 从 HTML 提取 <input id="ticket" value="hex串">
      → 变换规则：最后4字符移到最前
        → 计算 token 签名（见上）
        → GM_xmlhttpRequest POST /api.ashx
           参数: method=getHistoryTrend&key=京东URL&t=时间戳&token=MD5
           Header: Cookie + Authorization: BasicAuth <变换后ticket> + Content-Type
           期望响应: {"code":0, "data":{"datePrice":"[ts,price,''],..."}}
            → ECharts 渲染（series.data = [[ts, price, note], ...]）
```

## 图表渲染

参考原页面 `echartsTrend.js`（`https://mmbres.manmanbuy.com/pc_tool/echartsTrend.js`）。

数据格式：`[[timestamp, price, note], ...]`

ECharts option 要点：
- xAxis: `type: 'time'`, `boundaryGap: false`
- yAxis: `type: 'value'`, 动态计算 min/max
- series: `type: 'line'`, data 为 `[ts, price, note]` 三元组
- tooltip 显示日期、价格、优惠备注
- markLine 标记最高/最低价

## 如何测试

1. 访问 `https://tool.manmanbuy.com` 确保已登录
2. F12 → Application → Cookies，复制 `tool.manmanbuy.com` 域名下的所有 Cookie
3. 打开京东商品页（`https://item.jd.com/xxxx.html`）
4. 点击右侧浮动按钮 → 在弹窗粘贴 Cookie → 保存
5. F12 → Console 看 `[jd-ph]` 前缀日志
